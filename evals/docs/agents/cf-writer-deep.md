# Evaluating cf-writer-deep

## What We Test

Whether the cf-writer-deep agent (sonnet model) produces higher-quality technical documentation with nuanced analysis compared to both the lightweight cf-writer agent and baseline Claude. We measure depth, nuance handling, technical accuracy, and token efficiency.

## How Agents Are Tested

The cf-writer-deep agent is dispatched by skills that need to generate nuanced technical content where depth and reasoning matter. It uses the sonnet model for higher quality output than the haiku-based cf-writer.

## Rubric Criteria

| Criterion          | Weight | Description                                                                |
| ------------------ | ------ | -------------------------------------------------------------------------- |
| depth              | 0.30   | Goes beyond surface-level explanation to provide deep technical insight    |
| nuance_handling    | 0.25   | Handles complex topics with appropriate nuance rather than oversimplifying |
| technical_accuracy | 0.25   | Technical claims and code examples are correct                             |
| token_efficiency   | 0.20   | Depth is achieved efficiently without unnecessary verbosity                |

### Scoring Levels

**depth** (automated check: section depth >= 3):

- 0: Content is shallow -- could be generated by a quick web search summary
- 1: Covers basics adequately but does not explore nuances or edge cases
- 2: Good depth on main topic but shallow on important subtopics
- 3: Thorough treatment with nuanced analysis of trade-offs, edge cases, and implications

**nuance_handling**:

- 0: Oversimplifies complex topics -- presents false certainty on debatable points
- 1: Acknowledges complexity exists but does not explore it meaningfully
- 2: Handles most nuances well but oversimplifies one important area
- 3: Correctly identifies gray areas, presents multiple perspectives, avoids false dichotomies

**technical_accuracy**:

- 0: Contains technical errors that would mislead a reader
- 1: Mostly correct but one technical claim is wrong or a code example has a bug
- 2: Technically accurate but some explanations could be more precise
- 3: All technical claims correct, code examples work, explanations are precise

**token_efficiency** (automated check: word count):

- 0: Extremely verbose -- repeats the same points in different words
- 1: Some repetition and filler that could be cut
- 2: Mostly efficient but a few passages are unnecessarily verbose
- 3: Every section adds unique value -- depth achieved through substance, not volume

Notes: Writer-deep uses sonnet for higher quality reasoning. Quality bar is higher than cf-writer. Shallow output should score lower here than it would for the lightweight writer.

## Evaluation Method

### Indirect (via skills)

The cf-writer-deep agent may be dispatched by skills requiring nuanced technical documentation. Like cf-writer, it was not explicitly observed in Wave 1 or Wave 2 eval runs. Skills like cf-research generate technical reports, but it is unclear whether they dispatch the writer-deep agent or handle content generation directly.

### Direct (if applicable)

No direct cf-writer-deep evaluation was performed. To test directly:

1. Provide a complex technical writing task (e.g., "Write a technical analysis of cache invalidation strategies with trade-offs")
2. Measure output against the rubric
3. Compare sonnet writer-deep output vs haiku writer output vs direct Claude output
4. The key comparison is depth and nuance, not just accuracy

## What We Compare

- **cf-writer-deep (sonnet)**: Expected to produce deeper analysis, better handling of nuance, more thorough treatment of trade-offs
- **cf-writer (haiku)**: Expected to produce adequate but shallower documentation at lower cost
- **Baseline Claude (no agent)**: Expected to produce good documentation inline, without the structured agent workflow

The key question is whether the agent's sonnet-based reasoning produces measurably better technical documents than what the main model produces directly. Since the main model in evals was already opus, the writer-deep agent (sonnet) might actually produce lower-quality output than the calling context.

## Reliability Assessment

- **Sample size**: 0 (agent was never evaluated)
- **Confidence**: None
- **Known issues**: Same as cf-writer -- the agent was never observed in any eval run, and it is unclear which skills dispatch it. Additionally, since the eval model is opus, dispatching a sonnet-based agent may actually reduce quality, which would be counterintuitive.
- **Recommendation**: The cf-writer-deep agent has not been evaluated. To evaluate it:
  1. Create technical writing tasks that specifically require depth and nuance
  2. Compare writer-deep (sonnet) output vs writer (haiku) output on the same tasks
  3. Measure the quality/cost trade-off between the two writer agents
  4. Investigate whether dispatching a lower-tier model (sonnet) from a higher-tier context (opus) is actually beneficial or harmful
  5. Identify which skills dispatch writer-deep and instrument those runs
