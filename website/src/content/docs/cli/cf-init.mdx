## Overview

Interactive setup wizard for Coding Friend. Run it in the root directory of your project:

```bash
cf init
```

The wizard runs each step linearly. Steps that are already configured show `[done]` and are skipped automatically. You can run `cf init` as many times as you want.

To edit individual settings later, use [`cf config`](/docs/cli/cf-config/) instead.

## Config File Locations

At the start, `cf init` displays config file locations:

```
Config files:
  Global: ~/.coding-friend/config.json
  Local:  .coding-friend/config.json (this project)
Local overrides global at property level.
```

## Status Display

Before the wizard starts, current configuration is shown with scope labels:

```
Current configuration:
  [both]   docsDir (my-docs)
  [global] Docs language (en)
  [-]      /cf-learn config
```

| Label      | Meaning                                  | Color  |
| ---------- | ---------------------------------------- | ------ |
| `[global]` | Set only in global config                | blue   |
| `[local]`  | Set only in local (project) config       | green  |
| `[both]`   | Set in both global and local config      | cyan   |
| `[-]`      | Not configured anywhere (using defaults) | yellow |

## Setup Steps

### Step 1: Docs folder name

Sets the top-level folder name for all Coding Friend outputs (default: `"docs"`). Must be a single folder name, not a path (no slashes). After entering the name, you choose whether to save it globally or locally.

The documentation subfolders are created automatically alongside the docs folder:

- `{docsDir}/plans/` -- Implementation plans and task breakdowns
- `{docsDir}/memory/` -- Project knowledge and reference docs
- `{docsDir}/research/` -- Research findings and deep dives
- `{docsDir}/learn/` -- Learning notes and extracted knowledge
- `{docsDir}/sessions/` -- Session sync files

**Folder rename behavior:**

- Save to **local**: if the old folder exists, it is renamed to the new name; otherwise the new folder is created.
- Save to **global**: if local config already has `docsDir` set, no folder changes are made (the project manages its own). Otherwise, the existing global folder is renamed (or a new one is created).

### Step 2: Configure `.gitignore`

Choose whether to add Coding Friend artifacts to `.gitignore`:

- **Yes, ignore all** -- adds all folders
- **Partial** -- pick which folders to ignore
- **No** -- keep everything tracked

This step has no scope question (`.gitignore` is always project-local).

```bash
# >>> coding-friend managed
docs/plans/
docs/memory/
docs/research/
docs/learn/
docs/sessions/
.coding-friend/
# <<< coding-friend managed
```

### Step 3: Docs language

Sets the language for generated docs (plans, memory, research, ask). Choose **English** (`en`), **Vietnamese** (`vi`), or **Other** (enter a custom name). After choosing, select global or local scope.

### Step 4: `/cf-learn` config

Configures all `/cf-learn` settings in sequence:

1. **Language** -- language for learning notes
2. **Output dir** -- local (`docs/learn/`) or external path
3. **Categories** -- use defaults, keep current, or customize
4. **Auto-commit** -- only shown for external dirs with git
5. **README index** -- none, single, or per-category

After configuring all fields, you choose **one** scope for the entire learn object.

### Step 5: Shell completion

Auto-detects your shell (zsh/bash) and adds tab completion for `cf` commands. No scope question (global by nature).

### Step 6: Statusline

Interactive checklist to pick which components to show in the [statusline](/docs/cli/cf-statusline/).

### Step 7: Claude permissions

Only appears when using an **external** output directory for `/cf-learn`:

- Adds `Read`, `Edit`, `Write` permission rules to `~/.claude/settings.json`
- If auto-commit is enabled, also adds `git add` / `git commit` rules

## Cancelling

In steps that use a selection menu, a "Cancel init" option is available at the bottom. Selecting it exits the wizard immediately.

## Non-Git Directories

`cf init` can work outside git repos. The `.gitignore` step is automatically skipped in that case.
