## Overview

Interactive setup for Coding Friend. You can run it as many times as you want; only pending steps are offered. Run following command in the root directory of your project.

```bash
cf init
```

## Setup Steps

`cf init` scans your current configuration and presents a checklist of up to 7 steps:

1. Create documentation folders
2. Add Coding Friend artifacts to `.gitignore`
3. Set docs language (plans, memory, research, ask)
4. Configure [`/cf-learn`](/docs/skills/cf-learn/) settings (includes language)
5. Set up shell tab completion
6. Configure [Claude Code statusline](/docs/cli/cf-statusline/)
7. Add Claude Code permission rules (conditional)

Each step shows `[done]` or `[pending]`. You then choose either `Apply all pending`, `Pick which to apply`, or `Cancel`.

### Create docs folders

Following folders will be creater in your project:

- `docs/plans/` — Implementation plans and task breakdowns
- `docs/memory/` — Project knowledge and reference docs
- `docs/research/` — Research findings and deep dives
- `docs/learn/` — Learning notes and extracted knowledge

### Configure `.gitignore`

In case you don't want CF's stuffs in your project, this option helps you ignore them.

```bash
# >>> coding-friend managed
docs/plans/
docs/memory/
docs/research/
docs/learn/
.coding-friend/
# <<< coding-friend managed
```

### Set docs language

Sets the language for generated docs in the `docs/` folder (plans, memory, research, ask). Choose **English** (`en`), **Vietnamese** (`vi`), or **Other** (enter a custom name). Saved as the top-level `language` property in config.

### Configure [`/cf-learn`](/docs/skills/cf-learn/)

This step walks you through the `/cf-learn` configuration, starting with its language setting. The language for `/cf-learn` notes is separate from the docs language above — choose **English** (`en`), **Vietnamese** (`vi`), or **Other**. Saved as `learn.language` in config. If not set, falls back to the docs language above, then English.

After your configs, following settings will be configured. It's can be in `config.json` ([global or local](/docs/configuration/config-json/)), depends on where you choose to save the config.

```json
{
  "learn": {
    "language": "en",
    "outputDir": "docs/learn",
    "categories": [
      {
        "name": "concepts",
        "description": "Design patterns, algorithms, architecture principles"
      },
      {
        "name": "patterns",
        "description": "Repository pattern, observer pattern"
      },
      {
        "name": "languages",
        "description": "Language-specific features, syntax, idioms"
      },
      { "name": "tools", "description": "Libraries, frameworks, CLI tools" },
      { "name": "debugging", "description": "Debugging techniques, bug fixes" }
    ],
    "autoCommit": false,
    "readmeIndex": false
  }
}
```

| Field         | Type                        | Default        | Description                                                                            |
| ------------- | --------------------------- | -------------- | -------------------------------------------------------------------------------------- |
| `language`    | `string`                    | `"en"`         | Language for cf-learn notes. Falls back to top-level `language`, then `"en"`           |
| `outputDir`   | `string`                    | `"docs/learn"` | Where learning docs are saved. Absolute or `~/` paths for external storage             |
| `categories`  | `array`                     | 5 defaults     | Each entry has `name` and `description`                                                |
| `autoCommit`  | `boolean`                   | `false`        | Auto-commit after each `/cf-learn` (external dir only)                                 |
| `readmeIndex` | `boolean \| "per-category"` | `false`        | `true` for single README containing all indexes or `"per-category"` for one per folder |

You can edit this file directly at any time — no need to re-run `cf init`.

### Configure shell completion

Adds tab completion for `cf` commands to `~/.bashrc` or `~/.zshrc` (auto-detected)

### Configure statusline

Interactive checklist to pick which to show in the [statusline](/docs/cli/cf-statusline/).

```
cf v0.3.0 | MyProject | Opus 4.6 | ⎇ main | ctx 42% | 15% → 02:30
```

### Configure Claude Code permissions

- Only appears when using an **external** output directory for [`/cf-learn`](/docs/skills/cf-learn/)
- Adds `Read`, `Edit`, `Write` permission rules to `~/.claude/settings.json`
- If auto-commit is enabled, also adds `git add` / `git commit` rules

## Saving Configuration

Configuration can be saved globally (`~/.coding-friend/config.json`), locally (`<your-project>/.coding-friend/config.json`), or both.

See [Configuration](/docs/configuration/config-json/) for the config file format.

## Non-Git Directories

`cf init` can work outside git repos. In that case, the git-related steps (`.gitignore`, auto-commit, branch statusline) are automatically skipped.
