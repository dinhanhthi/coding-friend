## Installation

```bash
npm i -g coding-friend-cli
```

Check with `cf --version`. The binary is available as `cf`.

## Updating the CLI

```bash
# Update everything (plugin + CLI + statusline)
cf update

# Update only the CLI
cf update --cli
```

The CLI also **automatically checks for updates** after each command (cached every 24 hours). When a new version is available, it displays a notification and auto-updates in place. No manual action needed — the next time you run any `cf` command, the latest version will be used.

You can also update manually via npm:

```bash
npm install -g coding-friend-cli@latest
```

## Commands

| Command                                     | Purpose                                                                           |
| ------------------------------------------- | --------------------------------------------------------------------------------- |
| [`cf install`](/docs/cli/cf-install/)       | Install the Coding Friend plugin (supports `--user`/`--project`/`--local`)        |
| [`cf uninstall`](/docs/cli/cf-uninstall/)   | Remove the Coding Friend plugin (supports `--user`/`--project`/`--local`)         |
| [`cf disable`](/docs/cli/cf-disable/)       | Disable the plugin without uninstalling (supports `--user`/`--project`/`--local`) |
| [`cf enable`](/docs/cli/cf-enable/)         | Re-enable a disabled plugin (supports `--user`/`--project`/`--local`)             |
| [`cf init`](/docs/cli/cf-init/)             | Initialize workspace with documentation folders and platform configuration        |
| [`cf config`](/docs/cli/cf-config/)         | Manage Coding Friend configuration interactively                                  |
| [`cf host`](/docs/cli/cf-host/)             | Build and serve learning docs as a local website                                  |
| [`cf mcp`](/docs/cli/cf-mcp/)               | Setup MCP server for integrating learning docs with other LLM clients             |
| [`cf statusline`](/docs/cli/cf-statusline/) | Configure the Claude Code statusline with project info                            |
| [`cf update`](/docs/cli/cf-update/)         | Update plugin, CLI, and statusline (supports scope flags)                         |
| [`cf dev`](/docs/cli/cf-dev/)               | Switch between local and remote plugin for development                            |
| [`cf session`](/docs/cli/cf-session/)       | Save and load Claude Code sessions across machines                                |
| `cf help`                                   | Show help information                                                             |

## Tab Completion

Tab completion is auto-configured during installation. You can immediately use `cf <command>` with completion support in your shell. Coding Friend will modify your shell configuration files to enable completion.
