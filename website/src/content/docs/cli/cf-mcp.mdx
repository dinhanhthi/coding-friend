## Overview

`cf mcp` generates [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) server configurations for both **learning documentation** and **memory system**. Use this to integrate with other LLM clients like ChatGPT, Claude web, or other AI tools.

## Usage

```bash
cf mcp [path]
```

The optional `path` argument overrides the learning docs folder (default: `<project>/docs/learn/`).

## Stale config detection

When `cf mcp` runs, it automatically checks whether `.mcp.json` in the current directory has a stale `coding-friend-memory` entry — i.e., one that points to an absolute path that no longer exists (common after updating Node.js or reinstalling the CLI).

- **Stale path (broken):** prints a warning and suggests running `cf memory mcp` to fix it
- **Old format but still valid:** prints a notice suggesting you migrate to the version-stable format
- **Already using `npx` format:** silent (no output)

## Output

The command prints JSON configuration for two MCP servers — Learn MCP and Memory MCP — for easy copy/paste into your MCP client config. It no longer lists the individual tools and resources to avoid terminal clutter; see [Available tools & resources](#available-tools--resources) below.

### Learn MCP

Configuration for the learning documentation server. Provides tools to search, read, create, and manage learning docs. See [/cf-learn](/docs/skills/cf-learn/#mcp-integration) for more details.

```json
{
  "mcpServers": {
    "coding-friend-learn": {
      "command": "npx",
      "args": [
        "-y",
        "coding-friend-cli",
        "mcp-serve-learn",
        "/path/to/your/project/docs/learn"
      ]
    }
  }
}
```

### Memory MCP

Configuration for the project memory server. Provides tools to store, search, list, update, and delete project memories. See [Memory System](/docs/reference/memory-system/#mcp-integration) for more details.

```json
{
  "mcpServers": {
    "coding-friend-memory": {
      "command": "npx",
      "args": [
        "-y",
        "coding-friend-cli",
        "mcp-serve",
        "/path/to/your/project/docs/memory"
      ]
    }
  }
}
```

## Available tools & resources

### Learn MCP tools

**Read:**

| Tool              | Description                                |
| ----------------- | ------------------------------------------ |
| `list-categories` | List all categories with doc counts        |
| `list-docs`       | List docs, filter by category/tag          |
| `read-doc`        | Read full content of a doc                 |
| `search-docs`     | Full-text search across all docs           |
| `get-review-list` | Docs that need review                      |

**Write:**

| Tool              | Description                                                 |
| ----------------- | ----------------------------------------------------------- |
| `create-doc`      | Create new learning doc                                     |
| `update-doc`      | Append content or update tags                               |
| `improve-doc`     | Get improvement suggestions                                 |
| `track-knowledge` | Record understanding level (remembered/needs-review/new)    |

### Memory MCP tools

| Tool              | Description                        |
| ----------------- | ---------------------------------- |
| `memory_store`    | Store a new memory                 |
| `memory_search`   | Search memories (keyword match)    |
| `memory_retrieve` | Get a specific memory by ID        |
| `memory_list`     | List memories with filtering       |
| `memory_update`   | Update existing memory             |
| `memory_delete`   | Delete a memory                    |

### Memory MCP resources

| Resource         | Description          |
| ---------------- | -------------------- |
| `memory://index` | Browse all memories  |
| `memory://stats` | Storage statistics   |

When `cf memory mcp` writes to `.mcp.json`, it uses the **version-stable `npx` format** — no hardcoded paths that break on CLI updates:

```json
{
  "mcpServers": {
    "coding-friend-memory": {
      "command": "npx",
      "args": [
        "-y",
        "coding-friend-cli",
        "mcp-serve",
        "/path/to/docs/memory"
      ]
    }
  }
}
```

This invokes `cf mcp-serve` via npx, which resolves the current globally-installed CLI at runtime instead of relying on a fixed path.

**Important:** The memory path is project-specific (points to `docs/memory/` in your project). Always use a **local** `.mcp.json` per project — do not add it to the global `~/.claude/.mcp.json`, as it will only work for one project.
