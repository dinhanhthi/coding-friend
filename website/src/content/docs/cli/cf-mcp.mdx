## Overview

`cf mcp` registers and configures the two Coding Friend [MCP](https://modelcontextprotocol.io/) servers. See [MCP Servers](/docs/reference/mcp/) for the full reference on both servers, their tools, and configuration options.

## Usage

```bash
cf mcp
```

The learn folder is always read from your global config (`~/.coding-friend/config.json`), defaulting to `~/.coding-friend/learn/`. To change it, run `cf config` → Learn settings → Output dir.

## What it does

1. **Auto-registers Learn MCP** (user scope) — runs `claude mcp add --scope user coding-friend-learn ...` so the server is available across all your projects. Skipped if already registered or if `learn.disabled` is set.
2. **Prompts for Memory MCP** — if the current project doesn't have `coding-friend-memory` in `.mcp.json`, offers to add it.
3. **Prints JSON config** for both servers — for copy/paste into non-Claude-Code MCP clients (Claude Desktop, Cursor, ChatGPT, etc.).
4. **Runs health checks** for both servers and prints fix hints inline.

## Stale config detection

When `cf mcp` runs, it automatically checks whether `.mcp.json` in the current directory has a stale `coding-friend-memory` entry — i.e., one that points to an absolute path that no longer exists (common after updating Node.js or reinstalling the CLI).

- **Stale path (broken):** prints a warning and suggests running `cf memory mcp` to fix it
- **Old format but still valid:** prints a notice suggesting you migrate to the version-stable format
- **Already using `npx` format:** silent (no output)
