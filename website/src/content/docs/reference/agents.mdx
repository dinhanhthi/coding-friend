## Overview

Coding Friend includes 5 specialized agents, each optimized for specific engineering workflows. Agents are automatically selected based on the skill you invoke.

## Agent Reference

| Agent           | Model           | Purpose                                                  |
| --------------- | --------------- | -------------------------------------------------------- |
| `code-reviewer` | Claude Opus 4.6 | Multi-layer code review with deep reasoning              |
| `implementer`   | Claude Opus 4.6 | TDD-driven implementation with test-first approach       |
| `planner`       | Claude Opus 4.6 | Exploration, task breakdown, and implementation planning |
| `writer`        | Claude Haiku    | Lightweight document writing and markdown generation     |
| `writer-deep`   | Claude Opus 4.6 | Deep reasoning for nuanced technical documentation       |

## When Each Agent Is Used

### code-reviewer

Invoked by [`/cf-review [target]`](/docs/skills/cf-review/). The agent handles **execution** — reading the diff, reading full files for context, and formatting the report.

The review **methodology** (4-layer checklist, security vulnerability table, confidence scoring) lives in [`cf-auto-review`](/docs/skills/cf-auto-review/), which the agent loads at review time. This keeps methodology in one place rather than duplicated across components.

```
/cf-review (workflow orchestration)
    ↓ fork + dispatch
code-reviewer (execution — how to review)
    ↓ loads
cf-auto-review (methodology — what to check)
```

### implementer

Dispatched by [`/cf-plan [task]`](/docs/skills/cf-plan/), [`/cf-fix [bug]`](/docs/skills/cf-fix/), and [`/cf-optimize [target]`](/docs/skills/cf-optimize/) for test-first implementation. Also dispatched by [`cf-tdd`](/docs/skills/cf-tdd/) for substantial implementations (3+ files).

The agent runs in an isolated context and follows strict RED → GREEN → REFACTOR discipline. Skills pass it a detailed prompt with task context, file paths, and test patterns, then review the agent's report.

```
/cf-plan (clarify → explore → plan → approve)
    ↓ dispatch per task
implementer (RED → GREEN → REFACTOR → report) × N tasks

/cf-fix (diagnose → confirm approach)
    ↓ dispatch
implementer (write regression test → fix → verify → report)

/cf-optimize (baseline → analyze → plan)
    ↓ dispatch
implementer (write test → optimize → verify → report)

cf-tdd (auto-invoked for substantial work)
    ↓ dispatch
implementer (RED → GREEN → REFACTOR → report)
```

### planner

Invoked by `/cf-plan [task]`. Breaks down complex tasks into implementation steps with exploration.

### writer

Used for quick document generation, skill documentation, and lightweight markdown files.

### writer-deep

Invoked by `/cf-learn [topic]` with deep reasoning enabled. Extracts nuanced knowledge and creates comprehensive technical documentation.
