## Overview

Configure Coding Friend behavior using `.coding-friend/config.json`. This file lets you customize documentation directories, enable/disable hooks, and adjust learning extraction settings.

## Config File Locations

- **Global**: `~/.coding-friend/config.json` -- applies to all projects
- **Local**: `.coding-friend/config.json` -- project-specific settings

## How to Edit

You can edit configuration in two ways:

1. **Interactive**: Run `cf config` for a menu-driven editor, or `cf init` during first-time setup
2. **Manual**: Edit the JSON files directly at the paths above

## Configuration Schema

### Documentation Folder Name

```json
{
  "docsDir": "docs"
}
```

The name of the top-level folder where all skill outputs are stored (default: `"docs"`). This is a **folder name**, not a path -- it must be a single name relative to your project root, with no slashes. All skill output subfolders sit inside it: `plans/`, `memory/`, `research/`, `learn/`, and `sessions/`.

### Hooks

Control which automatic hooks run during your session:

```json
{
  "hooks": {
    "privacyBlock": true,
    "scoutBlock": true,
    "devRulesReminder": true,
    "contextTracker": true
  }
}
```

All hooks default to `true`. Set to `false` to disable.

### Commit Settings

```json
{
  "commit": {
    "verify": true,
    "conventionalCommits": true
  }
}
```

- `verify`: Run tests before allowing commit (default: `true`)
- `conventionalCommits`: Enforce conventional commit format (default: `true`)

### Learning Extraction

```json
{
  "learn": {
    "language": "en",
    "outputDir": "docs/learn",
    "categories": [
      {
        "name": "concepts",
        "description": "Design patterns, algorithms, architecture principles"
      },
      {
        "name": "patterns",
        "description": "Repository pattern, observer pattern"
      },
      {
        "name": "languages",
        "description": "Language-specific features, syntax, idioms"
      },
      { "name": "tools", "description": "Libraries, frameworks, CLI tools" },
      { "name": "debugging", "description": "Debugging techniques, bug fixes" }
    ],
    "autoCommit": false,
    "readmeIndex": false
  }
}
```

| Field         | Type                        | Default        | Description                                                                            |
| ------------- | --------------------------- | -------------- | -------------------------------------------------------------------------------------- |
| `language`    | `string`                    | `"en"`         | Language for cf-learn notes. Falls back to top-level `language`, then `"en"`           |
| `outputDir`   | `string`                    | `"docs/learn"` | Where learning docs are saved. Absolute or `~/` paths for external storage             |
| `categories`  | `array`                     | 5 defaults     | Each entry has `name` and `description`                                                |
| `autoCommit`  | `boolean`                   | `false`        | Auto-commit after each `/cf-learn` (external dir only)                                 |
| `readmeIndex` | `boolean \| "per-category"` | `false`        | `true` for single README containing all indexes or `"per-category"` for one per folder |

## Example Config

```json
{
  "docsDir": "docs",
  "devRulesReminder": true,
  "hooks": {
    "privacyBlock": true,
    "scoutBlock": true,
    "devRulesReminder": true,
    "contextTracker": true
  },
  "commit": {
    "verify": true,
    "conventionalCommits": true
  },
  "learn": {
    "language": "en",
    "outputDir": "docs/learn",
    "categories": [
      {
        "name": "concepts",
        "description": "Design patterns, algorithms, architecture principles"
      },
      { "name": "tools", "description": "Libraries, frameworks, CLI tools" }
    ],
    "autoCommit": false,
    "readmeIndex": false
  }
}
```

## Layered Configuration

Coding Friend supports two configuration levels:

1. **Global**: `~/.coding-friend/config.json` -- applies to all projects
2. **Local**: `.coding-friend/config.json` -- project-specific settings

Local settings override global settings **at the property level** (top-level key merge, not deep merge). For example, if global has `{ "language": "en", "docsDir": "docs" }` and local has `{ "language": "vi" }`, the result is `{ "language": "vi", "docsDir": "docs" }`. But if local has a `learn` object, it replaces the entire global `learn` object.

## Custom Skill Guides

Separate from `config.json`, you can extend built-in skills with project-specific guidance. See [Custom Skill Guides](/docs/configuration/custom-skill-guides) for full documentation and examples.
