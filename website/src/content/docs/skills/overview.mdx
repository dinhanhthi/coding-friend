Coding Friend provides a comprehensive toolkit of 15 skills that automate common engineering workflows. Eleven are triggered on-demand via slash commands, while four activate automatically in response to your coding patterns.

## Slash Commands

Interactive skills you trigger directly to solve specific problems.

| Command                                | Description                                              | Example                                      |
| -------------------------------------- | -------------------------------------------------------- | -------------------------------------------- |
| [`/cf-plan`](/skills/cf-plan/)         | Brainstorm and create structured implementation plans    | `/cf-plan Build a REST API with auth`        |
| [`/cf-fix`](/skills/cf-fix/)           | Quick bug fix workflow with problem verification         | `/cf-fix Login fails with 401 error`         |
| [`/cf-ask`](/skills/cf-ask/)           | Quick Q&A about your codebase with persistent memory     | `/cf-ask How does the auth middleware work?` |
| [`/cf-optimize`](/skills/cf-optimize/) | Structured optimization with before/after measurement    | `/cf-optimize getUserById query`             |
| [`/cf-review`](/skills/cf-review/)     | Multi-layer code review in a forked subagent             | `/cf-review src/auth/`                       |
| [`/cf-commit`](/skills/cf-commit/)     | Smart conventional commits with diff analysis            | `/cf-commit refactor auth flow`              |
| [`/cf-ship`](/skills/cf-ship/)         | Verify, commit, push, and create PR in one command       | `/cf-ship Add notification system`           |
| [`/cf-remember`](/skills/cf-remember/) | Capture project knowledge for AI memory across sessions  | `/cf-remember auth flow decisions`           |
| [`/cf-learn`](/skills/cf-learn/)       | Extract human learning docs from coding sessions         | `/cf-learn TypeScript patterns`              |
| [`/cf-research`](/skills/cf-research/) | In-depth research with web search and parallel subagents | `/cf-research best practices for JWT tokens` |
| [`/cf-help`](/skills/cf-help/)         | Answer questions about Coding Friend                     | `/cf-help how does cf-tdd work?`             |

## Auto-Invoked Skills

These skills activate automatically based on what you're doing, without needing an explicit command.

| Skill                                         | Activates When                  | What It Does                                                                                      |
| --------------------------------------------- | ------------------------------- | ------------------------------------------------------------------------------------------------- |
| [`cf-tdd`](/skills/cf-tdd/)                   | Writing new code                | Enforces test-driven development: RED → GREEN → REFACTOR                                          |
| [`cf-sys-debug`](/skills/cf-sys-debug/)       | Debugging issues                | Guides 4-phase systematic debugging: Observe, Hypothesize, Test, Fix                              |
| [`cf-auto-review`](/skills/cf-auto-review/)   | Reviewing code changes          | Applies multi-layer review checklist (correctness, security, performance, maintainability, tests) |
| [`cf-verification`](/skills/cf-verification/) | Before claiming task complete   | Ensures tests pass, changes verified, no regressions introduced                                   |
| [`cf-learn`](/skills/cf-learn/)               | After substantial new knowledge | Auto-extracts learnings to docs/learn/ for future reference                                       |

## Custom Skill Guides

You can extend any built-in skill with your own guidance by creating a `<skill-name>-custom/` directory with a `SKILL.md` file in `.coding-friend/skills/` (per-project) or `~/.coding-friend/skills/` (global). Each guide supports 3 optional sections:

- **`## Before`** — runs before the builtin workflow starts
- **`## Rules`** — additional rules applied throughout the workflow
- **`## After`** — runs after the final step completes

For example, `.coding-friend/skills/cf-commit-custom/SKILL.md` can add custom commit rules or trigger tests after each commit. See [Custom Skill Guides](/docs/configuration/custom-skill-guides/) for full details.
