Context footprint: `⚡⚡` (medium) — [what does this mean?](/docs/reference/context-usage/)

The `/cf-ask` skill answers questions about your codebase and saves the answers to persistent memory. Future sessions can reference these answers, building institutional knowledge.

## Usage

```bash
/cf-ask [question]
```

## What It Does

1. **Recalls from Memory** — Searches existing `docs/memory/` using a 3-tier frontmatter grep (description → tags → full-text) for previously answered questions or related context
2. **Analyzes Codebase** — If not already answered, explores relevant files via the [`cf-explorer`](/docs/agents/cf-explorer/) agent
3. **Provides Answer** — Gives clear, concise explanation with code examples
4. **Saves to Memory** — Delegates to [`cf-writer`](/docs/agents/cf-writer/) agent, stores Q&A in `docs/memory/` with frontmatter (title, description, tags) for efficient future recall

## Examples

```bash
/cf-ask How does the auth middleware work?
```

## When to Use

- Onboarding to a new codebase
- Understanding design decisions
- Locating specific functionality
- Quick clarifications during development

## Memory Recall

Before exploring the codebase, `/cf-ask` searches existing memory using a **3-tier grep** on frontmatter fields:

1. **Tier 1 — Descriptions** (cheapest): Grep `description:` fields for question keywords
2. **Tier 2 — Tags**: Grep `tags:` fields if no description match
3. **Tier 3 — Full-text**: Last resort content search

If a direct match is found, the existing answer is presented — no redundant exploration needed.

## Memory System

Answers are saved to `docs/memory/` with YAML frontmatter for efficient recall:

```yaml
---
title: "Auth Flow"
description: "JWT auth with refresh tokens and OAuth2 integration"
tags: [auth, jwt, oauth]
created: 2026-03-05
updated: 2026-03-05
---
```

Organized by topic:

```
docs/memory/
  ├── features/
  │   ├── auth-flow.md
  │   └── api-endpoints.md
  ├── conventions/
  │   ├── error-handling.md
  │   └── state-management.md
  └── decisions/
      ├── jwt-vs-sessions.md
      └── database-choice.md
```

## Benefits

- **Reduces Ramp-Up Time** — Captured knowledge speeds onboarding
- **Fast Recall** — Frontmatter grep finds relevant docs without reading full content
- **Searchable** — Find answers from previous conversations
- **AI-Readable** — Helps AI understand your codebase faster
