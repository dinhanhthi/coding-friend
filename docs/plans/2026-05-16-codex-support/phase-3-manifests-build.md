2. **3.2 Codex plugin manifest scaffold**
   - Files: `plugin-codex/.codex-plugin/plugin.json` (new — minimal placeholder; full file is generated by build script in 3.3)
   - Content: stub with `name: "coding-friend"`, `version: "{{VERSION}}"`, pointers `skills: "./skills/"`, `hooks: "./hooks/hooks.json"`, `mcpServers: "./.mcp.json"`. Build script in 3.3 substitutes version.
   - Verify: `codex plugin marketplace add /Users/thi/git/coding-friend` succeeds against a temporary `CODEX_HOME`. Note: `codex plugin marketplace upgrade coding-friend-marketplace` is Git-marketplace-only in Codex CLI v0.130.0 and rejects local marketplaces; local artifact verification uses `marketplace add`.
   - Rollback: `rm -rf plugin-codex/`.

3. **3.3 `scripts/build-codex-plugin.js`** (new)
   - Files: `scripts/build-codex-plugin.js` (new), `scripts/__tests__/build-codex-plugin.test.ts` (new)
   - Responsibilities:
     - Read `plugin/package.json` (or root) → version string
     - Copy `plugin/skills/`, `plugin/agents/` into `plugin-codex/`
     - Transform every `{{cf:slash NAME}}` → `$NAME`; `{{cf:agent_ref NAME}}` → `$NAME`; `{{cf:skill_invoke NAME}}` → `load $NAME`; `{{cf:plugin_root}}` → `${PLUGIN_ROOT}`; `{{cf:dispatch ...}}` → natural-language equivalent (template literal); `{{cf:host}}` → `Codex CLI`
     - Transform Claude-only hook event references in `hooks.json` (skip `TaskCreated`/`TaskCompleted` entries when generating Codex hooks.json — keep `PreCompact` and `SubagentStart`/`SubagentStop`, which Phase 0 verified Codex supports). Drop `async` fields because Codex hooks run synchronously. Build a per-host event allowlist.
     - Convert each `plugin/agents/cf-*.md` to `plugin-codex/agents/cf-*.toml` using frontmatter `name`+`description` → TOML keys; markdown body → `developer_instructions` string. (Phase 6 owns the converter; this task just calls it.)
     - Write `plugin-codex/.codex-plugin/plugin.json` with stamped version
     - Idempotent: rerunning produces byte-identical output (sort JSON keys, fix line endings)
   - Verify: `node scripts/build-codex-plugin.js` produces `plugin-codex/` with no diff on second run; test asserts placeholder transforms; test asserts version stamping.
   - Rollback: `rm -rf plugin-codex/`; delete script.

4. **3.4 Wire npm scripts**
   - Files: `package.json` (edit — add `"build:codex": "node scripts/build-codex-plugin.js"` and `"verify:codex-drift": "npm run build:codex && git diff --exit-code plugin-codex/"`)
   - Generate the initial `plugin-codex/` artifact and commit it
   - Verify: `npm run build:codex` + `git status` shows no diff after initial generation; `npm run verify:codex-drift` exits 0
   - Rollback: revert package.json; `rm -rf plugin-codex/`

## Exit criteria

- `.agents/plugins/marketplace.json` exists
- `plugin-codex/` exists with full skills/hooks/agents/manifest, generated by `npm run build:codex`
- `codex plugin marketplace add ./coding-friend` (local) lists `coding-friend` plugin
- `npm run verify:codex-drift` passes
- Build script is idempotent
